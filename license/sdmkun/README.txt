shiroi danmakukun ver.1.5.8

・概要

シューティングの弾避けの練習にと思って作ったものです。
300種類以上の弾幕が楽しめます。
弾幕定義言語BulletMLで簡単なスクリプトを書けば自分で弾幕を作れます。

・コンパイル、インストール

Windows の場合

実行ファイルは sdmkun.exe です。
ダブルクリックして下さい。

setup.exe を実行すると拡張子が .bml となっているファイルを
ダブルクリックするなり、ブラウザから直接実行すると、
すぐにその弾幕がプレイできるようになります。
その際 user ディレクトリ以下にそのファイルはコピーされているので、
次回以降普通に実行した場合もメニューに現れます。

ちなみに setup.exe は関連付けを行なっているだけです。
解除は適宜手動でやって下さい。

Unix系の場合

> make
でコンパイルできると思います。
src ディレクトリが圧縮されてますが、
bzip2 が入っていれば勝手に解凍します。
> ./sdmkun [xml or bml or lua file]
で実行できます。

・操作方法

カーソルキーの上下左右で移動。
z キーで移動速度をダウン・決定。
x キーでキャンセル。
エスケープキーでリセット。
PageUp と PageDown でカーソルがたくさん動きます。

ジョイスティックは、
一個目のボタンが z に、二個目が x 、
三個目が ESC にそれぞれ対応してます。
あと、七個目と八個目でページアップダウンです。

キーバインドは save/key.conf を変更すれば変更できます。
んなもんわからんわ、っていう人は下記に設定ツールがあるんでお試しあれ。

パッケージ内の mkconf.html
http://user.ecc.u-tokyo.ac.jp/~s31552/wp/sdmkun/mkconf.cgi

・設定項目

たくさん設定項目を設けました。
いろいろいじって遊んでみて下さい。
わかりにくいところを少し説明します。

処理が遅い人は fps を落とすか、文字飾りを off にして下さい。

スペシャルってのは適切な名前が思いつかなかったんですが、
だんだんレベルが上がっていくモードです。
セレクト-メインモード-スペシャルon でランキング右がわに記載されます。
こいつを選択すると他の弾幕ごとのオプションは無効化されます。

画面サイズが変わるのが欝陶しければ大きい画面で固定できます。

スクリーンショットの先読みを on にすると、
メモリ使用量が増えますが、セレクトモードの動作が速くなると思います。

万が一設定項目をいじっていると動かなくなったりした場合は、
新しくダウンロードしなおして、
save/conf ファイルを上書きして下さい。
気に入った設定での save/conf ファイルを残しておくと良いかも。

・オリジナル弾幕の作りかた

user ディレクトリに入れた拡張子が ".xml" または ".bml" のファイルは
セレクトモードの項目に現われるようにしました。
BulletMLの書き方は、BulletMLの本家、
http://www.asahi-net.or.jp/~cs8k-cyu/bulletml/
がチュートリアルになっています。
その他 bosses.d 内のファイルもサンプルになるでしょう。
まあ、HTMLが書ける位の人で、
シューティング好きならぜひ書いてみて下さい。

bml ファイルというのは弾幕名などの情報を
弾幕自身に埋めこむためにでっちあげた形式です。
具体例は user/test.bml をどうぞ。

LuaBullet という新しい弾幕記述言語が参戦しました。
詳しくは LuaBullet.txt にて。

・使っているもの紹介。

Bulletsmorph
ABAさん作の弾幕生成エンジン。
なかなか面白い弾幕を作ってくれますが、
弾が多すぎになりがちなのが少し残念。
弾幕作成者のブレインストーミングには良いかと。

SDL
クロスプラットフォームなゲームライブラリ。
とにかく抜群の信頼性。自分のプログラムに集中できます。
低レベル過ぎるのがちと最初は酷だけど、
慣れるとサブライブラリの選択が楽しいのもまた魅力。

SDL_kanji_f1b
SDL_ttf は大袈裟すぎるので。
詳しくはこれの中の README を参照。

libbulletml
BulletML 流行ってほしいなあと作ってみたライブラリ。

tinyxml
XMLパーサ。Xerces はおおげさ過ぎ。他にもいろいろあるけど、
これが一番ちっちゃくてお手軽な感じでしたのでセレクト。

Lua
プログラムの中に組込まれることを想定してデザインされている
プログラミング言語。一晩で弾幕記述言語ができました。

boost
c++ 準標準ライブラリ。
java みたいな Object.toString が無いと不便？
boost::lexical_cast<std::string>
で終わりじゃん？という感じ。
全部挿入すると巨大なので使うものだけ置いています。

なんかサブライブラリがんがんなのがいい感じ。
再利用度高そうな雰囲気を醸しだしている、気がする。
